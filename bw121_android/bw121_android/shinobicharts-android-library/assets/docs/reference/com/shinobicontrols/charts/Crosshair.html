Class Overview

The Crosshair provides a small circle target with lines that extend to the axis. This is accompanied by a tooltip which displays the X and Y values the crosshair is currently locked to.

The crosshair is enabled with a tap-and-hold gesture and will lock to the nearest series to pan through the values. On a LineSeries the values will be interpolated between data points, on all other series types the crosshair will jump from data point to data point.

In order for a crosshair to be displayed on a chart series, the series must enable it by calling setCrosshairEnabled(boolean) with true. This defaults to false, so you will need to call this method on each series in your chart.

public Object getUserXValue ()

Returns the corresponding user value for this crosshair's current position on the X-axis. This will be null if the crosshair is not currently tracking a series.

The crosshair does not know the type of this value so returns it as an Object. However, it will be of the corresponding type for the X-axis and so can be safely cast e.g. for a NumberAxis it can be cast to a Double, for a DateTimeAxis it can be cast to a Date.

Note that for a CategoryAxis, which is an extension of the NumberAxis, the returned value will be a Double. This number represents the index in the getCategories() list of the corresponding category String.

Returns
  • the corresponding user value for this crosshair's current position on the X-axis or null if it is not currently tracking a series

public boolean isActive ()

Whether this crosshair is currently active. The crosshair may still be active even if it is not currently shown on the plot area. For example, zooming the chart with the crosshair displayed may result in the crosshair being moved past the edge of the plot area. As it is still active a swipe gesture will move it back inside the plot area.
